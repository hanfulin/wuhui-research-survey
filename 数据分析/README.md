# 数据分析流水线（模板先行）

> ⚠️ **科研诚信红线**：本目录的"模拟数据"**仅用于在真实问卷回收前把分析流程和统计表模板搭好**，
> **绝不能**写入课题报告、专题报告或《原创性声明》覆盖的任何成果。
> 结题报告中的数据必须是问卷星导出的**真实作答**。

## 它解决什么

真实数据还没收齐，但分析代码、统计表、图表口径可以**现在就做好并验证**。
真实数据一到，**一行命令换输入**即可跑出真结果，不用临时赶代码。

## 三个文件

| 文件 | 作用 |
|------|------|
| `wenjuan_schema.py` | 问卷 41 题的结构定义（题型/分卷/选项），生成与分析共用 |
| `生成模拟数据.py` | 造**测试用**模拟数据，显著标注、不可入报告 |
| `分析.py` | 读 CSV → 去标识化 → 分卷 → 频数/量表/交叉 → 出统计表+摘要 |

## 怎么用（无需懂编程，复制命令即可）

### 阶段一：现在——用模拟数据把模板跑通

```
py 数据分析/生成模拟数据.py --a 70 --b 18 --c 25 --seed 42
py 数据分析/分析.py --input "数据分析/输出/模拟数据_测试用_请勿用于报告.csv"
```

打开 `数据分析/输出/分析摘要.md` 看效果。`--a/--b/--c` 是三类卷份数，按本院实际预估改。

### 阶段二：真实数据回收后——只换输入

1. 问卷星后台 → 「分析&下载」→ 导出**CSV**（不是 Excel）
2. 存为 `数据分析/真实数据.csv`
3. 跑：
   ```
   py 数据分析/分析.py --input "数据分析/真实数据.csv"
   ```
4. **分析代码一个字都不用改**——表头按题号匹配，问卷星题号不变即可

## 去标识化（关键纪律，已内置）

`分析.py` 会自动删除 `来自IP / 来源 / 地区 / 经纬度 / openid` 等可定位列，
只保留序号、提交时间、所用时间和答案。导出的统计表均为汇总值，无个人信息。

## 输出物（`数据分析/输出/`，不入 git）

- `分析摘要.md`：总览（回收量、各题频数、量表均值、交叉）
- `01_回收量.csv`、`02_量表题统计.csv`、`03_量表x职称.csv`、`04_量表x年限.csv`
- `QNN_*.csv`：每道单选/多选题的频数表

> 输出目录是临时结果，已在 `.gitignore` 排除；脚本本身入库版本管理。
