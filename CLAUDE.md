## 项目背景

- **项目性质**：2026 年江苏省卫生健康经济管理研究课题，《基于内部控制视角的公立医院科研经费包干制落地研究》；课题单位南通大学附属医院；研究期限 2026.04—2026.11（8 个月），**已立项**，**基金编号 CW202611**；GitHub 仓库 `wuhui-research-survey`
- **相关方/干系人**：负责人吴慧（会计师，财务与运营管理处）；课题组 6 人——施小如（高会·财务管理）、张煦利（会计师·成本运营）、缪惠芝（会计师·薪酬）、严娟（助会·预算）、丁海霞（高会·内控）、姜素珍（高会·财务管理）；**技术搭建由韩付林（GitHub `hanfulin`）支持**（吴慧零基础）；受访对象＝本院三类岗位 A 科研人员 / B 科研管理 / C 财务审计
- **起因与诉求**：国家推科研经费包干制改革（放权、增经费使用自主权）。但公立医院医教研交织、资金多元、内控要求严，包干制落地**不是简单减审批，而是放权基础上围绕关键风险点重构管理体系**。现有研究多停留政策解读/单环节，缺以内控为主线、同时覆盖风险识别＋控制环节＋负面清单＋分级授权＋信息化监管的系统成果。本课题以本院为对象，回答"为什么落地难 / 风险集中在哪 / 如何把放权嵌入内控 / 如何形成可执行可监督可推广的方案"
- **核心研究问题（一句话）**：如何通过问卷与访谈系统收集本院包干制落地现状，并从内部控制视角识别关键风险点、提出落地路径
- **红线与约束**：见下「关键纪律」（去标识化、绝不数据造假、申请书原件不改、敏感材料不入公开 repo、在线服务先验国内可达性）

## 研究目标与验收口径（申请书 2.2）

1. 系统把握落地主要矛盾（制度难点/管理堵点/风险点）——口径：形成本院现状诊断
2. 识别关键风险点与控制薄弱环节——口径：风险清单 + 用本院数据定级
3. 构建内控分析框架——口径：COSO 五要素 × 科研经费全生命周期框架成稿
4. 提出兼顾规范、效率、防控的落地路径与制度建议——口径：负面清单 + 分级授权 + 配套机制建议稿

## 主要研究内容（申请书 2.3 四方面）

1. **落地现状**：问卷 + 访谈 + 制度资料，摸清适用范围、审批报销、职责分工、部门协同、制度执行现状
2. **风险与控制薄弱环节**：围绕授权审批、支出边界、预算调整、结余资金管理、过程留痕、监督问责识别风险点
3. **基于内控的关键环节**：适用范围界定、负面清单设置、分级授权标准、流程优化、部门协同
4. **配套机制**：信息化嵌入式监管、科研财务专员、培训宣传、容错纠错

## 研究方法（绑研究对象/样本）

- **文献研究法**：最终采用 16 篇清单（非申请书原 14 篇，出入留痕见决策日志 2026-05-18）
- **问卷调查法**：A 科研 / B 管理 / C 财务三类，问卷星 41 题，去标识化，按"部门×职称×年限区间"统计

- **专家咨询法（Delphi）**：对负面清单、分级授权、控制要点多轮咨询至收敛

## 关键约定/术语表

- **包干制**：科研经费包干制，提高科研人员经费使用自主权的制度安排
- **内部控制 / COSO 五要素**：控制环境·风险评估·控制活动·信息与沟通·内部监督
- **负面清单**：经费不可用事项清单；**分级授权**：按金额/事项分级的审批授权标准
- **嵌入式监管**：把控制规则嵌进信息系统，事中自动管控
- **Delphi 法**：专家多轮匿名咨询至意见收敛
- **去标识化**：只存部门 / 职称级别 / 工作年限区间，绝不存姓名/工号/手机号
- **受访三类**：A 科研人员 / B 科研管理 / C 财务审计（问卷按此分卷）
- **单一数据源**：研究进度只写 `研究进展日志.md`，其他地方用链接指过去
- **核心论点（综述定稿）**：公立医院科研经费包干制的落地难题，本质不是"放还是控"的二元选择，而是**放权之后内控如何重构的系统设计问题**；内部控制由此从"约束工具"转向"落地支撑"。——2026-05-30 v1 定稿，2026-05-31 v2 巩固；后续论文/报告/总报告一律以此论点为主轴，不再回退到 enabler 等抽象洋词
- **《现代医院》投稿包装口径**：稿件不能按"普通文献综述"投，应按"文献研究 / 政策文本分析 / 医院管理框架构建"呈现；题名弱化"综述"，突出内控逻辑、风险识别、管理路径/框架构建，摘要结果段和第 7 节实践启示必须对应这一口径。——2026-06-02 复核定

## 关键纪律

1. **去标识化**：数据库/文件绝不存受访者真实姓名/工号/手机号，只存部门、职称级别、年限区间
2. **绝不数据造假**（最硬红线）：模拟数据仅用于把 `数据分析/` 模板跑通、四重标注"测试用·禁入报告"，**绝不冒充真实样本、绝不进《原创性声明》覆盖的任何成果**；回收不足走近普查口径 + 访谈/Delphi 补强，不靠编数
3. **单一数据源**：研究进度只写 `研究进展日志.md`，其他地方用链接指过去
4. **申请书原件不改**：2026.03 已申报立项，文献清单等以课题组最终清单为准（出入留痕见决策日志）
5. **在线服务选型先验国内可达性**：凡涉及受访者访问的服务，先确认国内秒开（Firebase/Pages 之鉴）
6. **commit 信息**：写"做了什么 + 为什么"，不写"更新"两个字
7. **回复语言**：所有交流使用简体中文
8. **大块产出立即落盘**：任何 ≥3000 字的单日大块产出（论文/章节/方案稿），完工**当下立即 Write 到仓库 .md**，不只暂存对话——Claude 不同对话间无记忆，对话窗口一关就丢；收工 Skill 自检"本次有无大块文本只停在对话里"（踩坑：5-30 综述 v1 6850 字险些丢失，见踩坑记录）
9. **国际制度/具体数字双源验证**：综述/报告引用国际制度年份、期刊卷期、页码等具体数字时，永远 web search 一手来源（机构官网为先）+ URL 双源印证，**不靠记忆**；事实错误最易出在"宏观计划 vs 子计划试点"两层时间错位（踩坑：ERC Lump Sum 错记 2021 实为 2024，见踩坑记录）
10. **Synology Drive 路径下绝不 `git init`**：项目目录 `E:\SynologyDrive\02_项目\吴慧课题\` 是 Synology 同步副本，**无 `.git`**（同步盘默认不同步 `.git`）。push 走带 git 检出的另一台机器；在同步路径下 init 会污染同步、与另一台机器双 checkout 冲突
