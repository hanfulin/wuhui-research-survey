# 问卷星搭建指南

> 用途：把定稿的问卷搭到「问卷星」上，生成二维码供受访者扫码匿名填答，再导出数据做统计。
> 状态：操作指南 v1 · 2026-05-19
> 背景：Firebase / GitHub Pages 国内访问不稳定，受访者扫码大概率打不开，故改用问卷星（国内主流、医护熟悉、扫码秒开）。
> 数据源单一：题目以《问卷题目初稿》定稿 v2 为唯一蓝本；本仓库 `survey.html` 仅留作**离线预览参考**（团队可本地打开直观看到问卷长相、培训调查员用），**不作数据收集**。

---

## 〇、纪律红线（搭建时务必照做）

1. **去标识化**：全卷不加姓名、工号、手机号、身份证、具体科室名等任何可定位题；背景题只到"岗位类别/职级区间/年限区间"为止（即定稿 T1–T4，不许加细）。
2. **匿名设置**：问卷星后台关闭一切会暴露身份的采集项（见第五步）。
3. **E3 访谈意愿**：选"愿意"者**不在本问卷里留联系方式**（那会破坏匿名）。文案已写明"届时另行匿名联络"，照此执行。
4. **数据单一蓝本**：题目、选项、字段顺序一律以《问卷题目初稿》定稿 v2 为准，不在问卷星上随手改词；如需改，先改定稿 md 再同步。

---

## 一、账号与新建

1. 浏览器打开 https://www.wjx.cn ，注册/登录（手机号即可，免费版够用）
2. 顶部「创建问卷」→ 选「**调查/评估**」类型
3. 问卷标题填：**公立医院科研经费包干制落地情况调查问卷**
4. 进入编辑器

## 二、首页知情同意（必做）

问卷星没有独立"首屏勾选"，用以下组合实现等效效果：

1. **问卷说明**：编辑器「问卷标题」下方有"问卷说明/欢迎语"，把《知情同意与匿名承诺文案》"问卷版-知情同意书"全文粘进去。
2. **加一道必答确认题**作为第 1 题：
   - 题型：**单选**
   - 题干：`我已阅读并理解上述说明，自愿参与本次匿名问卷调查`
   - 选项：① `同意，开始填写`　② `不同意`
   - 设为**必答**
   - 设**跳转逻辑**：选②"不同意" → **跳转到问卷结束**（问卷星：该选项 → 结束答题）
3. 这样未同意者无法进入正式题目，等效强制勾选。

## 三、按定稿录入题目（题型对照）

严格按《问卷题目初稿》定稿 v2 顺序录入。题型映射：

| 定稿标注 | 问卷星题型 | 备注 |
|---------|-----------|------|
| [单选] | 单选题 | 选项照抄定稿 |
| [多选] | 多选题 | T9/A4 等；B6 设"最多选 3 项"（多选题→高级→最多可选 3） |
| [量表] | 量表题（5 级李克特） | 统一：1 完全不符合 … 5 完全符合 |
| [开放] | 填空题（多行文本） | A7/A10/B9/C9/E2 设为**非必答**（选填） |

**录入顺序**（关键，分流逻辑依赖顺序）：
确认题 → 通用 T1–T9 → A1–A10 → B1–B9 → C1–C9 → 结尾 E1–E3

> 量表题方向核对：定稿待办里有"Likert 方向一致性"一项，录入时一并检查，避免正反向混杂。

## 四、分流逻辑（最关键——按 T1 岗位自动跳题）

定稿设计：T1 选什么岗位，就只答对应那一卷，跳过其余两卷。用问卷星「**跳转逻辑**」实现。

在 **T1（岗位类别）** 这道单选题设置跳转逻辑：

| T1 选项 | 跳转到 |
|---------|--------|
| ① 科研人员 | 跳到 **A1** |
| ② 科研管理部门人员 | 跳到 **B1** |
| ③ 财务/审计部门人员 | 跳到 **C1** |
| ④ 其他 | 跳到 **E1**（跳过 A/B/C，直接结尾） |

> ⚠️ 注意：T1 之后还有 T2–T9（通用题），所以严格说应在 **T9 之后**做分流。两种做法择一：
> - **做法一（推荐，简单）**：把分流跳转设在 **T9**——但 T9 是多选题不便做岗位跳转。改为：在 T1 与 T9 之间不跳，正常答完 T1–T9；在 **T9 设跳转**按"上一题 T1 的答案"分流不可行。
> - **做法二（实际采用）**：通用题 T1–T9 全员都答，**在 T9 后插一道隐藏分流不可行**——问卷星跳转只认本题选项。故最稳妥：**在 T1 直接做岗位跳转到各卷，把 T2–T9 通用题移到每卷开头重复，或把 T2–T9 也纳入"所有人先答"**。

**最终采用方案（清晰版）**：
1. T1 单独一页，作为分流闸。
2. T1 跳转：①→A 卷首题；②→B 卷首题；③→C 卷首题；④→结尾。
3. **把 T2–T9（通用背景+政策认知）放进每一卷的开头各放一份**（A 卷开头放 T2–T9，B 卷开头放 T2–T9，C 卷/其他同理）。
   - 或更省事：T2–T9 改放在 T1 之前？不行，T1 要先分流。
   - 推荐：A/B/C 三卷各自开头复制 T2–T9，字段名加前缀（如 a_t5、b_t5）便于合并统计。
4. 每卷答完后：A 卷末题 A10 设跳转 → **E1**；B 卷末题 B9 → **E1**；C 卷 C9 自然接 E1。
5. 结尾 E1–E3 所有人都答。

> 这是问卷星单选跳转的固有限制带来的工程取舍。**统计时按 t5/a_t5/b_t5/c_t5 合并为同一变量**即可，不影响分析。导出后用 Claude/Excel 合并很快。

## 五、匿名与防作弊设置（去标识化关键）

编辑器右上「设置」/「问卷设置」里：

- **答题设置**：关闭"记录答题者微信昵称/头像"、关闭"需要微信授权"
- **防重复**：可设"每个微信/每个 IP 限答 1 次"——⚠️ 取舍：限 IP 可能误伤同一科室共用网络的人。建议设**"每台设备限答 1 次"（按 cookie）**，对匿名影响最小。
- **不收集**：确认未开启任何"获取地理位置""手机号验证"等
- **来源**：可关闭"显示答题来源/IP 归属地"展示

## 六、生成二维码 + 发放

1. 编辑完点「完成编辑」→「发布问卷」
2. 「发送问卷」页 →「二维码」→ 下载二维码图片
3. 二维码图片可：印在通知/海报、发科室微信群、邮件附图
4. 把二维码图保存到本仓库 `问卷设计/` 或 Obsidian，便于团队取用

## 七、回收与数据导出（做统计）

1. 「分析&下载」→ 可看实时回收量与基础图表
2. **导出原始数据**：下载为 **Excel**（免费版支持）
3. 统计分析：交给 Claude / Excel 做（频数、交叉、量表均值等）——**不依赖问卷星付费的 SPSS 导出与高级分析**
4. 合并分流字段：导出后把 a_t5/b_t5/c_t5 等并回 t5 统一变量再分析
